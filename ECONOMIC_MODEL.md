## 1. 定位（不是工具，是「造工具的工作台」）

WorkBench 是一个 **AI-native 的工作坊**：普通人只要**开口说痛点**，`fde-copilot` 持续追问 + 替他调研分析，把真实场景提炼成技术规格；`loop-engineer` 据此把**专属工具**造出来；`capability-packs` 提供配置一次即可复用的**原子能力**。

- 服务对象：**个体 / 家庭 / 社区 / 企业**，只要会说话就行。
- 交付物是**工具**（在线工具 / 可安装应用 / 自动化 / agent），一次性产物只是最浅子集。
- 心态：**把自由度给用户**——帮每个人建立自己 AI-native 的生活、组织流程、商业 AI-native 化。

## 2. 三个阶段：造 · 部署 · 跑（互相独立）

| 阶段 | 是什么 | 在哪 |
|---|---|---|
| **造 Build** | fde-copilot 追问 → 规格 → loop-engineer 造出工具 | **一定能在 Mac Mini 工作坊上完成** |
| **部署 Deploy** | 把造好的工具放到能跑的地方 | 与「造」**分开**;我们给方案：Cloudflare / 自持 / 专属 Mac Mini |
| **跑 Run/Host** | 让工具持续运行、维护、接 token | 免费(Cloudflare+你的账号) 或 托管(按月/年) |

**关键：造 ≠ 部署 ≠ 跑。** 造完就是你的东西,可下载带走,部署与跑另算。

## 3. 分层上手（从完全自持到全托管）

> 核心全开源（Apache 2.0，数字公共物品）。**付费买的是便利/可靠/专业，不是准入。** 一句话：让普通人得到科技带来的数字生活福祉。

**① 自部署（免费，Apache 2.0 公共物品）**
- 拿仓库 + 部署指南 + 配置方式，**结合你自己的 Claude Code**，把这套 WorkBench 直接搬到你自己的电脑上跑——就是我们现在这种形式。见 [DEPLOY-PLATFORM.md](./DEPLOY-PLATFORM.md)（含仓库与三个子项目清单）。
- 你搭好、验证、持续维护提升这套 agent，这就是**你的** WorkBench。零费用。

**② 用我们部署的 WorkBench（不懂技术，按量租用）**
- 不会技术就直接用我们部署好的 web 版。后台接 API 有成本，**我们只收「能让这个公共物品可持续」的成本，不高额收费**。
- 像**公共物品租用 / 公交票 / 水费**。**按量计费（不是按时间）**。

**③ 造完工具后的部署（两条路）**
- **自部署**：按我们的自部署方案自己上。
- **我们部署**：① 完全部署在**商业服务器**；② 你**租一台 Mac Mini**、我们给你部署上去——这台就成你的**专属机**，数据全在你手里（含开机密码），租金较高，若干周期后**产权可转给你**。（专属机主要面向个人。）

**④ 企业**
- 长期租一台 workbench 持续完善自己的业务流程 / AI-native 化；外加
- **企业咨询服务**（类 FDE，我们叫 **ANE**，长期）：受时间精力所限，**$500 / 2 小时，每位用户每月一次**；更多需另行沟通（客户多了无法保证质量）。几次咨询后，你通常就有能力自己完成。

**可携带 = 数据主权（贯穿所有层）**：任何时候一键下载**全部**（对话、流程、规格、代码）→ 清空，或加密留存、下次再导入继续。主权活在「你的数据 + 部署」，「造」的算力可共享摊薄。

## 4. 社区协作（公共物品的乘数）

我们是**开放社区**，会吸引很多 **builder**。他们可以作为你的 consultant，和你**组队**——有能力的建设者 + 有行业经验的专家，一起用这个公共物品，帮你的组织/企业完成 AI-native 化。

- **目标**：我们提供公共物品，**任何有能力的人都能通过它互相协作、达成双方目标**。
- **我们的职责**：维护这个公共物品、提供交流环境与协作机会，以及给不懂技术的人的**基础按量租用**服务。
- 收益按 [PGL 两维度分账](#6-与-mycelium--pgl-的对接分账有两个正交维度)（产品 + 算力），对齐 Mycelium 的协作网络。

## 5. 计价基础：两个计量表（token + 计算单元）

产品各应用已**实时统计两种单位**（顶栏用量条）：

- **Token**：模型 API 消耗——「造」和「跑」里调模型的部分。免费托管用**你自己的 token/账号**;托管层可接 Workers AI 或我们提供的 API（另一服务层）。
- **计算单元 CU（墙钟计算秒）**：机器/电/时间——对应专属机器与长跑任务。

**结构建议**：token **透传/你自付**,CU/机器走**按月**（$1000/台）。两个成本驱动分开,不硬摊。

## 6. 与 Mycelium / PGL 的对接（分账有两个正交维度）

我上版用的分账逻辑，是**直接套 PGL 的 AgentStore 三角色**——但那只覆盖「谁造的」，不覆盖「谁的机器跑的」。你的设备共享（E）是**新增的一维**。理清成两层：

**维度一 · 产品价值（谁造的工具/能力）→ PGL 三角色**
- **Supplier 50–90%**（写核心能力的人，如 capability-pack 作者）
- **Wrapper 0–40%**（把它包装成普通人能用的）
- **Seller 固定 10%**（AgentStore 渠道）
- 商业收入 **40% 回流社区国库**。适用于工具/能力在 AgentStore 出售。

**维度二 · 算力/基础设施（谁的机器跑的）→ 设备共享分成（PGL 现有角色不覆盖，新增）**
- **设备提供者**：贡献 Mac Mini 到算力池，按跑的用量拿一份（对应 Mycelium「传输」范式——算力像养分在网络里流动）。
- **维护团队（公共物品）**：维护开源平台，拿一份。
- 可复用 Mycelium 的 **SuperPaymaster v5 角色/质押/惩罚体系** 做设备注册、信誉、结算，零新合约。

**一笔付费 = 产品价值部分（按维度一分给造工具的人）+ 算力部分（按维度二分给提供机器的人 + 维护团队）。** 两者组合、互不干扰。

> 咨询/托管走商业实体（对齐「开源组织 + HyperCapital」双生结构），产品核心留公共物品层。

## 7. 反馈与建议（中立客观）

> 你已解掉我上版的「共享vs专属」张力（两模式设计）。下面是对**refined** 模型的新观察。

1. **免费托管有天然边界,正好成清晰分层。** Cloudflare 免费层只接得住**简单/较无状态**的工具;要**本地模型 / 常驻 / 本地 FLUX** 的重工具接不住 → 天然导向专属 Mac Mini。**建议**：把「简单→免费 Cloudflare / 重→专属机」明确写成产品分层,别让用户对免费层期待过高。
2. **$1000/月 是「企业/发烧友」档,不是大众档。** 硬件~$600–1000,一个月回本,之后是电+运维+价值。对企业(替代几万块的雇人)很划算;对个人偏贵。**建议**：明确 **个人主走 A+免费C,企业主走 B+D**,别用一个价位盖所有人。
3. **托管层(非免费)必须把「维护人力」计进去。** 模型/依赖/平台 API 变化(如小红书 Cookie 过期)是持续人工。免费层因为用户自付 token/自担,可不含维护;但**托管层定价要显式含维护**,否则规模化亏损。
4. **「产权可转」是一种融资结构(租转售),要定清 N 和条款。** 涉及分期/所有权/数据清空的法律与账务,早点把周期、剩余价值、退出写清。
5. **全开源下护城河说清**：不在代码,在于 ① capability-packs 库**广度+策展**、② 场景累积让 fde-copilot **洞察越来越准**、③ 托管**可靠性**。可携带/可自持越彻底,越要靠这三点留住人。
6. **咨询是高毛利、与产品分账、且刻意「不 scale」**：**$500/2 小时·每用户每月一次**，保护稀缺人力;定位是**授人以渔**——几次后客户自己能干，剩下的交给社区 builder 组队接。走商业实体、40% 回流，别和产品经济混算。
7. **设备共享(E)是「传输」范式的收入化,但要防两类风险**：① 别人的 Mac Mini 跑**你客户的数据**,信任/隔离/加密边界要硬(敏感数据只落用户专属机、共享池只接非敏感/可擦除任务);② 设备质量/在线率参差,需信誉分 + SLA + 惩罚(可复用 Mycelium 的角色/质押/惩罚体系)。

## 一句话

**开源、数据主权、为普通人造专属 AI-native 套装的工作台。分层上手：① 自部署(免费,Apache 2.0) ② 不懂技术者按量租用我们部署的(只收可持续成本,似水费) ③ 造完自部署或我们部署(商业服务器 / 专属 Mac Mini) ④ 企业租用 + ANE 咨询($500/2h·每月一次,授人以渔)。上层有 builder 社区组队协作、设备共享算力池;token 你自付、CU 按机计;分账两维度——产品按 PGL 三角色、算力按设备共享分成。我们的职责:维护公共物品 + 提供协作环境 + 基础按量服务。**
