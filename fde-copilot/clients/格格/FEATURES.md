# Feature 细节

> 客户：格格 ｜ 创建：2026-07-15 ｜ 最近更新：2026-07-15 (R3)
> 每个 feature：用户故事 + 验收标准(Given/When/Then) + 边界/异常 + 优先级
> ✅ R3 参数补齐：地域=东北、时间线=2年并行、双交付。✅ R4：落点确认锁定、城市不限、科研成果到位（论文+ICDI奖+自研agent）、HiLinkup采用、新增 F8 视频生成。

## R4 锁定参数（贯穿各 Feature）
- **地域范围**：F2 抓取 = 东北三省（辽/吉/黑）全域，**不限城市**，不做城市级过滤。
- **领域模板**：F4 教案/作品集**锁定** **数字经济/电子商务/新媒体运营 × 创新创业** 模板（R4 客户已确认，其它学院模板不预置）。
- **王牌资产**：格格**自研"数字创新创业 agent"** + 1 篇论文 + ICDI 优秀学生奖 → 贯穿 F3 简历/F6 试讲/F4 教案案例/F8 视频的第一亮点。
- **生成三件套**：软件"一步到位"生成 **简历(F3) + 教案(F4) + 视频(F8)**。
- **模型接入**：统一走 **HiLinkup Gateway**（已确认），代码用 `HILINKUP_API_KEY` 环境变量抽象，key 位置待补不阻塞开发。
- **排期锚点**：≈2 年长跑，倒排到每年秋招（9–12 月）/春招（3–5 月）高职招聘季；M0 立即启动，M1 第 1 年完成弹药积累，M2 贴近毕业季前 6–9 个月进入投递高峰。
- **双交付**：软件内每类生成物均须支持"一键导出为材料清单包"（简历 docx/教案 docx/岗位表 xlsx/试讲稿/求职 checklist）。

## 交付路线（增量）
- **M0（诊断与地基）**：F1 竞争力画像 + F2 岗位情报底库。先让格格看清"我在哪、要去哪、差什么"。
- **M1（造弹药）**：F3 AI native 简历 + F4 作品集/教案生成。把"三无"补成"有作品"。
- **M2（打靶与临阵）**：F5 投递管理 + F6 试讲/面试演练。真正对接岗位、拿下面试。
- 每个 Feature 均可独立交付+验证。

### F1 · 竞争力画像与差距分析【M0 · P0】
**用户故事：** 作为求职中的博士，我想知道对照高职创新创业教师岗的真实要求，我现在差什么、按什么顺序补，这样我不盲目使劲。
**验收标准：**
- Given 格格已录入现状（学历/在读状态、博士研究方向、已有科研成果、语言、是否党员、年龄等），When 运行画像，Then 输出一份差距报告，含：① 满足项；② 缺口项（如"学历未认证""无双师材料""无教案"）；③ 每项的补齐动作与优先级；④ 对"泰国博士认可度"风险的应对建议。
- Given 差距报告已生成，When 格格补齐某项，Then 报告状态可更新并重算就绪度。
**边界/异常：** 岗位要求因校而异——报告需说明"基于通用高职创新创业岗画像"，具体岗以 F2 抓到的公告为准。
**依赖：** 需要 GAPS 中 Q 系列（研究方向/成果/身份）答案才能精准。

### F2 · 岗位情报聚合与匹配【M0 · P0】
**用户故事：** 作为求职者，我想只看到"博士友好、经验不限、接受海外博士"的高职创新创业岗，不想在硬门槛不符的岗上浪费时间。
**验收标准：**
- Given 已配置目标地域/意向，When 抓取招聘信息，Then 输出岗位列表，每条含：院校、岗位名、学历要求、经验要求、是否接受海外博士（若公告未写则标"待确认"）、截止日期、来源链接。
- Given 一条岗位，When 系统评估匹配度，Then 给出"匹配/存疑/不符"标签与理由（对照 F1 画像）。
**边界/异常：** 招聘信息源合规抓取；抓不到"是否接受泰博"时不臆断，标"待向院校人事确认"。抓取范围=东北三省（辽/吉/黑）；具体城市未定，先全域后按后续城市偏好收窄。编制/合同岗**均纳入**（客户都接受），不做身份过滤。
**量化阈值（R3）：** 东北全域累计沉淀"匹配/存疑"岗位 ≥30 个作为长跑底库；每个招聘季刷新。
**依赖：** 目标地域已定=东北（Q6 部分闭合，城市待定）。

### F3 · AI native 简历包装【M1 · P1】
**用户故事：** 作为无企业经历的博士，我想要一份把"清迈博士 + AI native 创新创业"讲成强故事的简历，弱化产业经历短板。
**验收标准：**
- Given 格格现状 + 目标岗，When 生成简历，Then 产出针对该岗定制的简历，突出：博士学术能力、**自研"数字创新创业 agent"作为头号亮点**、1 篇论文 + ICDI 优秀学生奖、AI 融合创新创业教学的差异化、可迁移能力；不虚构不具备的经历。
- Given 简历初稿，When AI 评审，Then 给出评分与逐条改进建议，可迭代至设定分数线。
**边界/异常：** 合规底线——不得编造企业/职业经历；"无经历"用真实可迁移能力与作品（自研 agent/论文/获奖）替代。
**验收断言（R4）：** 生成的简历正文必须包含"自研 agent"这一亮点条目（作为 AI native 实证），否则判不合格。

### F4 · 作品集与教案生成【M1 · P1】
**用户故事：** 作为没有教案的博士，我想快速拥有几份能拿得出手的创新创业教案/课程设计/AI 教学案例，作为试讲和作品集素材。
**验收标准：**
- Given 指定课题（如"大学生创新创业基础""创业计划书实训"）与二级学院场景（会计/旅游/制造，可选），When 生成，Then 产出结构完整的教案（教学目标/重难点/教学过程/AI 工具融入点/考核），可导出可编辑文档。
- Given 一份教案，When 格格要求调整，Then 可按反馈迭代。
**边界/异常：** 输出为"初稿+可编辑"，需格格本人审校确认为自己的作品（署名与原创责任在格格）。
**领域模板（R4·已锁定）：** 套 **数字经济/电子商务/新媒体运营 × 双创** 课题（如"数字经济创新创业基础""电商创业实训""新媒体创业项目路演指导""用 AI Agent 做创业机会识别"），贴合格格"数字创新"研究方向；**R4 客户已确认锁定**，其它学院模板（会计/旅游/制造）不预置。至少 1 份教案应以格格自研 agent 为教学案例。
**量化阈值（R3）：** M1 结束产出 ≥3 份完整教案/课程设计 + ≥1 套试讲稿，均经 AI 评审达设定分数线（见 F3/F6）。

### F5 · 投递管理与截止提醒【M2 · P2】
**用户故事：** 作为同时投多所院校的求职者，我想统一管理每个岗位的投递状态和截止日期，别错过。
**验收标准：**
- Given 从 F2 选中的岗位，When 加入投递看板，Then 可记录状态（待投/已投/初筛/试讲/面试/offer/拒），并对临近截止的岗位提醒。
**边界/异常：** 实际投递动作由格格本人完成，系统只做记录与提醒。

### F6 · 试讲/说课与面试演练【M2 · P1】
**用户故事：** 作为要试讲的候选人，我想有 AI 评委帮我练创新创业课试讲，并练熟"为什么泰国博士""无企业经历怎么教创业"这类必被问到的问题。
**验收标准：**
- Given 一份教案/课题，When 生成试讲稿并进入模拟，Then AI 扮演评委给出提问、逐项打分（教学设计/表达/AI 融合/临场）与改进建议。
- Given 高频问题库，When 格格作答，Then AI 给出话术优化（重点：泰博认可度、无产业经历、为何选高职）。
**边界/异常：** 演练为文本/语音辅助，不替代真人试讲临场表现。

### F7 · 材料清单包导出（双交付）【M1–M2 · P1】
**用户故事：** 作为要投多所院校的求职者，我想把系统里生成的所有资产一键打包成"可直接投递的材料清单包"，即使不打开软件也能用。
**验收标准：**
- Given 已生成简历/教案/岗位表/试讲稿，When 点击"导出材料包"，Then 产出一个结构化文件夹（含：定制简历 docx、教案作品集 docx/pdf、目标岗位清单 xlsx、试讲稿、**求职进度 checklist**、学历认证/双师型待办清单），命名规范、可离线交付。
- Given 岗位或资产更新，When 重新导出，Then 版本可区分、旧包不被静默覆盖。
**边界/异常：** 导出内容与软件内数据一致；材料包只是软件生成物的"落地快照"，不引入软件外的新事实。

### F8 · 说课/试讲视频生成【M1–M2 · P1】（R4 新增）
**用户故事：** 作为要交视频/要上试讲台的候选人，我想让软件把我的教案"一步到位"变成一段能录/能交的说课（或试讲）视频素材，而不是自己从零写脚本。
**验收标准：**
- Given 一份已生成的教案/课题（默认数字经济/电商×双创），When 点击"生成视频"，Then 产出**视频脚本 + 分镜/PPT 提纲 + 逐段讲解要点 + 时长配速（对齐招聘常见 10–20 分钟）**，可导出可编辑，并标注"AI 工具融入点"与"自研 agent 展示点"。
- Given 视频脚本初稿，When 格格要求调整，Then 可按反馈迭代；When AI 评审，Then 给出评分与改进建议。
**边界/异常：** **视频具体形态待 Q15 确认**（说课/试讲视频 / 自我介绍短视频 / agent 作品演示视频，可多选）——当前默认按"说课/试讲视频脚本+分镜"实现，若确认要真实视频合成再评估 TTS/数字人等能力。产出为"可录制脚本/素材"，实际出镜或录制由格格本人完成，系统不伪造格格本人影像。
**依赖：** 依赖 F4 教案产物；模型走 HiLinkup。**不阻塞 M0/M1 主线**，形态确认后细化。

**量化阈值汇总（R3/R4 已可定，下游可直接作测试断言）：**
- F2 东北岗位底库 ≥30 条；"匹配"岗均满足博士学历门槛。
- F3 简历 AI 评审目标分 **≥80/100**，未达标可迭代；正文必含"自研 agent"亮点条目。
- F4 作品集 ≥3 份教案 + ≥1 套课程设计，四要素齐全；≥1 份以自研 agent 为教学案例。
- F6 试讲模拟每次输出量化评分 + ≥3 条可执行改进点；高频问答覆盖"泰博认可度/无产业经历/为何高职/为何东北"四类必答题，并预置"你这个 agent 是怎么做的"技术追问。
- F8 视频脚本含分镜+讲解要点+时长配速，对齐 10–20 分钟；≥1 段脚本设置"自研 agent 展示点"。
- 排期：M0 即刻启动；M1 首年完成；M2 于毕业前 6–9 个月进入投递高峰（对齐秋招/春招季）。
